linux — это операционная система с терминалом, который позволяет полностью управлять многими процессами через команды, а именно создавать файлы, выполнять более сложные задачи и работать с проектами. это отличная платформа для AI-проектов. в отличие от Windows, Linux не использует привычное разделение на диски C: и D:, а работает с единой файловой системой.

терминал — это встроенный в Linux инструмент, через который выполняется большинство действий: от создания папок и файлов до разработки больших проектов. все действия выполняются с помощью команд или горячих клавиш.

bash — это оболочка, которая распознаёт команды, введённые разработчиком в терминале. она анализирует их, запускает в работу, а если команда написана с ошибкой, сообщает об этом.

git — это система контроля версий. очень важная система, которой пользуются многие разработчики. самое главное её преимущество — возможность работать над одним проектом всей командой одновременно. также Git помогает переносить проект на другой компьютер, хранит историю изменений, позволяет возвращаться к предыдущим версиям проекта и использовать ветки для экспериментов, не повреждая основную версию проекта.

github — это сайт, на котором хранятся удалённые репозитории. на этой платформе можно создать свой репозиторий, загрузить проект, хранить его в интернете и работать над ним вместе с другими разработчиками. в отличие от Git, GitHub работает через интернет.

docker — это программа и технология, которая позволяет запускать приложения в изолированных контейнерах. благодаря этому при переносе проекта на другое устройство для него создаётся такое же окружение со всеми необходимыми версиями программ и библиотек, поэтому приложение работает одинаково на разных компьютерах.

commit — это сохранённое состояние проекта в определённый момент времени. это не файл и не папка, а запись в истории Git, к которой при необходимости можно вернуться.

branch — это отдельная ветка проекта. она используется для экспериментов и разработки новых функций, чтобы не изменять основную версию проекта. если изменения получились удачными, ветки можно объединить в один проект.

repository — это папка проекта, в которой Git хранит файлы и историю их изменений. обычная папка после команды git init становится локальным Git-репозиторием. это полезно, когда нужно посмотреть историю изменений, понять, какие действия происходили с проектом, и при необходимости вернуться к предыдущей версии.

readme — это файл, который GitHub показывает сразу на главной странице репозитория. он помогает человеку быстро понять, о чём проект, для чего он создан и над чем работал разработчик. после прочтения README должно быть понятно, как устроен проект и для чего он нужен.


pwd — эта команда помогает определить, где ты сейчас находишься, а именно показывает текущую директорию (путь). она не показывает содержимое папки и её файлы

ls — эта команда позволяет узнать содержимое папки и посмотреть, какие файлы и папки в ней находятся

cd — перейти в нужную директорию. например, если мы находимся в ~/ai-internship/week-1/AI-Lab/linux-internship, то командой cd ~/ai-internship/week-1/AI-Lab мы переместимся в директорию AI-Lab

mkdir — создание новой папки

touch — создание нового файла

cp — это команда копирования. самое главное, правильно указать, что копировать и куда копировать, чтобы Bash правильно распознал команду и не возникла ошибка

mv — эта команда отвечает сразу за два действия. ей можно как переместить файл, так и переименовать его. также она позволяет выполнить оба действия одновременно

rm — этой командой можно удалить файл. для удаления папок используется параметр -r. также можно удалить несколько файлов одновременно, используя фигурные скобки {}

find — эта команда позволяет найти файлы и папки по различным условиям. например, можно использовать команду find . -maxdepth 2 -type f, чтобы найти все файлы внутри проекта

grep — поиск текста внутри файлов. можно искать как отдельные слова, так и даты или целые фразы

cat — показать содержимое файла

nano — эта команда позволяет открыть файл и написать в нём что-либо после создания командой touch. для сохранения используется Ctrl + O, затем Enter, а для выхода — Ctrl + X

chmod — изменение прав доступа к файлам и папкам. основные значения: 755, 644, 777

history — эта команда показывает историю ранее выполненных команд в терминале

man — эта команда открывает официальную инструкцию (справку) по выбранной команде Linux. например: man ls


git stash - временно сохраняет незавершённые изменения и возвращает рабочую папку к состоянию последнего коммита. то есть временно убирает изменения в специальное хранилище Git. после этого рабочая директория станет чистой, а изменения не потеряются. используется, когда срочно нужно исправить ошибку или изменить несколько файлов при незаконченной работе. в AI-проекте это может пригодиться, если ты меняешь код модели, но срочно должна проверить другую версию проекта или исправить файл конфигурации в основной ветке

git am - эта команда принимает патчи. патч - это файл, в котором записано, какие изменения были внесены в проект. главное отличие от обычного ручного копирования в том, что git am сохраняет автора и историю коммита, а не просто изменяет файлы. эта команда используется, когда разработчик получает изменения не через обычный git pull или pull request, а в виде patch-файла. команда чаще всего используется в крупных проектах. на донном этане она может пригодиться если я буду работать с Linux kernel

git archive - создаёт архив файлов из определённой версии Git-проекта. например, можно упаковать текущую версию проекта в .zip. команда берёт файлы из указанного коммита, ветки или тега и создаёт из них архив. используется, когда нужно отправить готовый проект  человеку, которому не нужна история Git. в будущем она может пригодиться, когда нужно: отправить проект без истории Git, создать архив готовой версии
